## Spike — 技术验证（P0 之前或并行）

### Spike 1：PixiJS v8 + Rive 集成验证
- 搭建最小 PixiJS v8 项目
- 集成 @rive-app/canvas，验证 Rive canvas 叠加或嵌入 PixiJS 的方案
- 加载一个测试 Rive 动画验证渲染 + 状态机切换
- **通过标准**：能正常渲染、状态切换流畅、和 PixiJS 场景层协调无冲突
- **失败方案**：改用 DragonBones 或退回序列帧

### Spike 2：PixiJS Canvas + Vue DOM 覆盖层通信
- PixiJS 渲染一个可点击热区
- 点击后通知 Vue 弹出一个面板
- Vue 面板关闭后通知 PixiJS 恢复
- **通过标准**：事件双向通信正常、无 z-index 冲突、触摸事件不穿透

### Spike 3：Camera 双模式（横屏全景 + 竖屏区域滑动）
- 实现 Camera zoom-in/out
- 横屏：正常 zoom-in 到区域
- 竖屏：区域级展示 + 左右滑动切换
- **通过标准**：两种模式切换流畅、resize 时自适应

### Spike 4：美术管线验证
- 用 AI（MidJourney/SD）生成一张区域特写底图
- 后期修整（去瑕疵、调色、裁切）
- 切片为 PixiJS 可用的精灵图
- 加载到 PixiJS 验证渲染效果
- **通过标准**：风格可接受、分辨率够用、流程可复制
- **失败方案**：调整 AI prompt 策略 或 寻找外包插画师

## P0 — 工程基座

### M0.1 工程初始化
- monorepo 搭建（pnpm workspace）
- client：Vue 3 + Vite + TypeScript
- server：NestJS + Prisma + SQLite
- shared：共享类型包
- ESLint + Prettier 配置
- Docker Compose（dev 环境）

### M0.2 PixiJS 场景基座 + 音频基座
- PixiJS v8 集成到 Vue 项目
- Canvas 全屏 + 响应式缩放（letterbox）
- 加载房间底图显示
- 自定义卡通光标
- Howler.js 集成 + audio sprite 加载框架
- Pinia + mitt 状态/事件架构搭建

### M0.3 Camera 系统（含移动端双模式）
- Camera 状态管理（zone、scale、pivot、transitioning）
- zoom-in / zoom-out 动画（GSAP 驱动 PixiJS 容器）
- 热区点击 → 触发 zoom-in
- 返回按钮 → zoom-out 回全景
- **横屏**：全景 + zoom-in
- **竖屏**：区域级展示 + 左右滑动切换 + 导航指示器
- 视口外精灵 renderable=false 性能优化

### M0.4 入口与密码
- 密码页 UI
- 后端密码校验 API（bcrypt）
- 资源加载期等待画面（门缝猫偷看 / loading 动画 + 随机小贴士）
- 开门过场动画（门推开 + 吱呀声）
- 认证态持久化（localStorage token）
- 移动端 Audio Context 在首次交互时解锁
- iOS DeviceMotion 权限在首次交互时申请
